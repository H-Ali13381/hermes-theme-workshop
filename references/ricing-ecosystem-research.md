## Part 1: Notable Dotfiles Repo Architectures

### Hyprland — Top Tier

#### prasanthrangan/hyprdots (HyDE) — 6k+ stars
- **Concept:** "Hypr Desktop Environment" — treats Hyprland as a full DE
- **Theme storage:** `~/.config/hyde/themes/<Name>/` — each folder contains wallpaper, Waybar CSS, Hyprland color overrides, Kitty colors, Rofi theme, cursor/icon names
- **Switch mechanism:** single `themeswitch.sh` reloads ALL components
- **Color engine:** Wallust auto-generates palettes from wallpaper
- **Pipeline:** `wallpaper → Wallust → palette.css/colors.sh → Waybar/Kitty/Rofi/Hyprland reload`
- **Extra:** themes are community-shareable as ZIP archives
- URL: https://github.com/prasanthrangan/hyprdots

#### end-4/dots-hyprland — 5k+ stars
- **Concept:** AGS (Aylur's GTK Shell) for all widgets written in TypeScript
- **Color engine:** matugen generates `material-colors.scss` + `material-colors.css` — EVERY widget imports these
- **Waybar CSS pattern:**
  ```css
  @import "colors.css";   /* generated by matugen */
  @import "base.css";     /* layout, sizing */
  @import "modules.css";  /* per-module styling */
  ```
- **Pipeline:** `wallpaper → matugen (Material You) → SCSS vars → AGS recompiles`
- URL: https://github.com/end-4/dots-hyprland

#### mylinuxforwork/dotfiles (ML4W) — 3k+ stars
- **Notable:** Only Hyprland setup with an actual theming **dashboard** (GTK4 settings app)
- **Config modularization via `source =` includes:**
  ```
  hyprland.conf
    source = conf/monitors.conf
    source = conf/keybinds.conf
    source = conf/animations.conf
    source = conf/windowrules.conf
    source = conf/colors.conf   ← only this changes per theme
  ```
- URL: https://github.com/mylinuxforwork/dotfiles

## Part 2: End-to-End Theming Tools

### Unified Color Propagation (the interesting layer)

#### matugen — Rust, 2023–2025, 1k+ stars ⭐ BEST MODERN CROSS-DISTRO
- Material Design 3 palette from a wallpaper (primary, secondary, tertiary, surface)
- Template engine uses `.mustache` files; `config.toml` maps templates to output paths
- Single command: `matugen image /path/to/wallpaper`
- Best modern pywal replacement; very active; Wayland-native community
- URL: https://github.com/InioX/matugen

#### tinty — Rust, active 2024–2025, 400+ stars ⭐ BEST NON-NIX BASE16 CLI
- Official CLI from tinted-theming org; spiritual successor to flavours
- Supports base16 (16 colors) AND base24 (24 colors)
- 100+ pre-built app templates; hook system runs reload signals after applying
- `tinty apply catppuccin-mocha` applies to all configured apps at once
- URL: https://github.com/tinted-theming/tinty

#### flavours — Rust, 1k+ stars (predecessor to tinty)
- Still functional; development slowed as tinty took over
- URL: https://github.com/Misterio77/flavours

#### wallust — Rust, 500+ stars
- Modern pywal replacement; faster extraction; TOML config; named palettes beyond 16 colors
- Can apply colorschemes WITHOUT a wallpaper
- URL: https://github.com/explosion-mental/wallust

#### pywal — Python, 8k+ stars, **ARCHIVED**
- The original. K-means on wallpaper pixels → 16-color palette → Jinja2 templates → reload
- Effectively replaced by wallust + matugen in modern setups; still widely referenced
- URL: https://github.com/dylanaraps/pywal

#### Stylix — Nix, 2k+ stars, explosive growth 2023–2025 ⭐ BEST OVERALL (NIX ONLY)
- NixOS/home-manager module: single base16 YAML or wallpaper → 50+ apps themed atomically
- Covers: GTK, Qt/Kvantum, terminals, WMs, bars, notifications, launchers, editors, browsers, GRUB, Plymouth, SDDM
- The most complete unified theming solution that exists
- URL: https://github.com/danth/stylix

### GTK / Qt Specific

#### Gradience — GUI, 1.5k+ stars
- GTK4/libadwaita color editor; edits Adwaita CSS custom properties; live preview
- Preset library (Catppuccin, Dracula, Nord); exports CSS snippets
- Best for theming GNOME/libadwaita apps without forking upstream themes
- URL: https://github.com/GradienceTeam/Gradience

#### wpgtk — GUI, 1.6k+ stars
- GTK frontend for pywal; generates actual GTK theme files from extracted colors (not just CSS vars)
- Activity slowed after 2022
- URL: https://github.com/deviantfero/wpgtk

#### nwg-look — GUI, 800+ stars
- lxappearance equivalent for Wayland / wlroots compositors
- GTK2/GTK3 theme, icon theme, cursor theme picker
- URL: https://github.com/nwg-piotr/nwg-look

#### Kvantum — 3k+ stars
- SVG-based Qt theme engine; used by essentially every KDE rice
- Most theme packs (Catppuccin, Nordic, gruvbox) ship both GTK and Kvantum variants
- URL: https://github.com/tsujan/Kvantum

## Part 3: Dotfile Managers and Color Propagation Architecture

### The Core Theming Problem
> "How do I change 50 app configs to use the same color palette without editing each one manually?"

---

### Solution Hierarchy (2026)

| Scenario | Tool Stack |
|----------|------------|
| NixOS power user | Stylix + home-manager flakes |
| Non-Nix, automated | dotfile manager (stow/chezmoi) + tinty for base16 propagation |
| Non-Nix, flexible/any format | chezmoi with `[data.colors]` in `chezmoi.toml` + `.tmpl` files |
| Wayland / Hyprland dynamic | matugen wallpaper pipeline + app reload hooks |

---

### Dotfile Managers (deployment layer, not theming engines)

#### GNU Stow
- Symlinks only; zero templating
- **Ricing pattern:** split configs into base file + colors include file; only re-stow the colors file per theme
- Theme switching: `stow -D themes/catppuccin && stow themes/gruvbox`

#### chezmoi — 13k+ stars ⭐ best non-Nix template approach
- Go `text/template` engine; `[data]` section in `chezmoi.toml` is a single source of truth
- `.tmpl` suffix files read `{{ .chezmoi.data.colors.background }}`
- `chezmoi apply` renders ALL templates simultaneously
- Machine-specific overrides via hostname detection in config template
- URL: https://github.com/twpayne/chezmoi

**Example `chezmoi.toml`:**
```toml
[data]
  [data.colors]
    background = "#1e1e2e"
    foreground = "#cdd6f4"
    accent     = "#89b4fa"
    red        = "#f38ba8"
    green      = "#a6e3a1"
  [data.font]
    monospace = "JetBrains Mono Nerd Font"
    size      = 12
```

**Example `.tmpl` usage:**
```toml
# alacritty.toml.tmpl
[colors.primary]
background = "{{ .chezmoi.data.colors.background }}"
foreground = "{{ .chezmoi.data.colors.foreground }}"
```

#### yadm
- Git wrapper for `$HOME`; no symlinks, actual files in home dir
- Alternate files: `alacritty.toml##class.dark` / `alacritty.toml##class.light`; switch with `yadm config local.class dark`
- Jinja2 templates available but limited for color injection
- Best for multi-machine setups, not fine-grained palette propagation
- URL: https://github.com/TheLocehiliosan/yadm

#### dotbot — Python, widely used
- YAML-driven installer (`install.conf.yaml`); `link`, `copy`, `shell` directives
- No templating built-in; use `shell` directive to call tinty/flavours for the color layer
- URL: https://github.com/anishathalye/dotbot

## Part 4: Base16 / Tinted-Theming Ecosystem

### Base16 Specification
16-color semantic palette (`base00`–`base0F`) defined in YAML; mustache templates render schemes into app configs.

| Slot | Semantic Role |
|------|--------------|
| base00 | background |
| base01 | lighter background (status bars, line numbers) |
| base02 | selection background |
| base03 | comments, invisibles |
| base04 | dark foreground (status bars) |
| base05 | default foreground |
| base06 | light foreground |
| base07 | light background |
| base08 | red — errors, deletion |
| base09 | orange — integers, constants |
| base0A | yellow — classes, search highlight |
| base0B | green — strings, success |
| base0C | cyan — regex, special characters |
| base0D | blue — functions, headings |
| base0E | magenta — keywords |
| base0F | brown — deprecated |

- Canonical org: https://github.com/tinted-theming
- Scheme library: https://github.com/tinted-theming/schemes
- Builder (Rust): https://github.com/tinted-theming/tinted-builder-rust
- 100+ app templates available

## Part 5: Key Implementation Patterns

### Wallpaper Daemon Pipeline (dominant Hyprland pattern)
```bash
#!/bin/bash
# wallpaper.sh <image>
swww img "$1" --transition-type grow
matugen image "$1"         # regenerates all templates
pkill -SIGUSR2 waybar      # reload CSS
pkill -SIGUSR1 kitty       # reload config
hyprctl reload             # hyprland colors
pkill dunst && dunst &     # restart dunst with new colors
```

### Modular Hyprland Config Pattern
```ini
# hyprland.conf (master)
source = ~/.config/hypr/conf/monitors.conf
source = ~/.config/hypr/conf/keybinds.conf
source = ~/.config/hypr/conf/animations.conf
source = ~/.config/hypr/conf/windowrules.conf
source = ~/.config/hypr/conf/colors.conf   # ← only this changes per theme
```

### Modular Waybar CSS Pattern
```css
/* style.css */
@import "colors.css";    /* generated by matugen/wal */
@import "base.css";      /* layout, sizing */
@import "modules.css";   /* per-module styling */
```
