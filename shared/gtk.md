# GTK Theming (GTK3/GTK4)

GTK is a completely separate rendering stack from Qt/KDE. Without this layer, a themed Qt desktop leaves all GTK apps (Firefox, Nautilus, GIMP) visually unchanged.

## What GTK Controls

- Window decorations and titlebar chrome for GTK3/GTK4 apps
- Button styles, entry fields, checkboxes, scrollbars
- Menu backgrounds and text colors
- NOT: web page content, app icons, font rendering

## settings.ini Format

Both `~/.config/gtk-3.0/settings.ini` and `~/.config/gtk-4.0/settings.ini` share the same format:

```ini
[Settings]
gtk-theme-name=Adwaita-dark
gtk-icon-theme-name=Papirus-Dark
gtk-font-name=Inter 11
gtk-cursor-theme-name=catppuccin-macchiato-teal-cursors
gtk-cursor-theme-size=24
gtk-toolbar-style=GTK_TOOLBAR_BOTH_HORIZ
gtk-button-images=1
gtk-menu-images=1
gtk-enable-event-sounds=0
gtk-enable-input-feedback-sounds=0
gtk-xft-antialias=1
gtk-xft-hinting=1
gtk-xft-hintstyle=hintslight
gtk-xft-rgba=rgb
gtk-application-prefer-dark-theme=1
```

## gtk.css Overrides (GTK4)

`~/.config/gtk-4.0/gtk.css` allows fine-grained color overrides with exact palette values:

```css
/* Generated by linux-ricing */
@define-color accent_color #7ad4f0;
@define-color accent_bg_color #7ad4f0;
@define-color accent_fg_color #0c1220;
@define-color destructive_color #cc3090;
@define-color success_color #2a8060;
@define-color warning_color #c87820;
@define-color error_color #cc3090;

window {
  background-color: #0c1220;
  color: #e4f0ff;
}

headerbar {
  background-color: #1c1e2a;
  color: #e4f0ff;
  border-bottom: 1px solid #7ad4f0;
}

entry {
  background-color: #1c1e2a;
  color: #e4f0ff;
  border: 1px solid #3d2214;
}

entry:focus { border-color: #7ad4f0; }

button {
  background-color: #1c1e2a;
  color: #e4f0ff;
}

button:hover {
  background-color: #7ad4f0;
  color: #0c1220;
}
```

## GTK Snapshot for Undo

```python
def snapshot_gtk_state() -> dict:
    state = {"gtk3_settings": None, "gtk4_settings": None, "gtk4_css": None}
    for key, path in [
        ("gtk3_settings", HOME / ".config/gtk-3.0/settings.ini"),
        ("gtk4_settings", HOME / ".config/gtk-4.0/settings.ini"),
        ("gtk4_css", HOME / ".config/gtk-4.0/gtk.css"),
    ]:
        if path.exists():
            state[key] = path.read_text(encoding="utf-8")
    return state
```

## Pitfalls

- **`kde-gtk-config` is NOT always installed.** Write `settings.ini` files directly.
- **GTK 3 and GTK 4 themes are separate.** A GTK3 theme won't auto-apply to GTK4 apps. Use `gtk.css` for GTK4.
- **GTK apps require restart** to pick up settings.ini changes — no live-reload equivalent.
- **`gtk-application-prefer-dark-theme=1`** only works for apps that respect it.
- **Adwaita-dark is always available** as a safe fallback — no install needed.
- **Firefox uses its own internal theming** on top of GTK chrome. Headerbar picks up GTK colors; page content does not.
