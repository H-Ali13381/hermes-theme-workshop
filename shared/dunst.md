# Dunst — Notification Daemon

Dunst handles desktop notifications on Wayland and X11. Uses INI config format with fragment includes. Works across compositors (Hyprland, Sway, i3, etc.).

---

## File Locations

```
~/.config/dunst/dunstrc            — main config
~/.config/dunst/hermes-dunstrc     — theme color fragment (included)
```

---

## Config Approach: INI Fragment Include

Dunst supports `include` directives (v1.7.0+). Write theme colors to a separate fragment and include it in the main dunstrc.

### Main dunstrc — add at the end:

```ini
include = ~/.config/dunst/hermes-dunstrc  # linux-ricing
```

### Theme fragment — hermes-dunstrc:

```ini
# Generated by linux-ricing — void-dragon

[global]
    frame_color = "#7ad4f0"
    separator_color = "#1c1e2a"
    font = JetBrainsMono Nerd Font 11
    frame_width = 2
    corner_radius = 0
    offset = 10x10
    origin = top-right
    transparency = 10

[urgency_low]
    background = "#0c1220"
    foreground = "#e4f0ff"
    frame_color = "#1c1e2a"
    timeout = 5

[urgency_normal]
    background = "#1c1e2a"
    foreground = "#e4f0ff"
    frame_color = "#7ad4f0"
    timeout = 10

[urgency_critical]
    background = "#cc3090"
    foreground = "#e4f0ff"
    frame_color = "#cc3090"
    timeout = 0
```

## Color Mapping from 10-Key Palette

| Dunst Setting | Palette Key | Context |
|---------------|-------------|---------|
| `[global] frame_color` | primary | Default notification border |
| `[global] separator_color` | surface | Divider between stacked notifications |
| `[urgency_low] background` | background | Subtle notifications |
| `[urgency_low] frame_color` | surface | Low-priority border |
| `[urgency_normal] background` | surface | Standard notifications |
| `[urgency_normal] frame_color` | primary | Standard border |
| `[urgency_critical] background` | danger | Urgent notifications |
| `[urgency_critical] frame_color` | danger | Urgent border |
| All `foreground` | foreground | Text color |

## Reload

```bash
pkill -HUP dunst
```

If dunst was started with a specific config path, you may need a full restart:

```bash
pkill dunst; dunst -config ~/.config/dunst/dunstrc &
```

## Dunst v2 Deprecated Keys (CRITICAL — generated configs often use these)

When generating dunst config fragments, NEVER use these v1 keys — they cause WARNING spam and are silently ignored in dunst v2:

### `geometry` — removed in v2

**Bad (v1):**
```ini
geometry = "380x5-20+44"
```

**Good (v2):**
```ini
width  = 380
offset = 20x44
origin = top-right
```

Decode the v1 geometry string: `WIDTHxMAX_NOTIFICATIONS-RIGHT_MARGIN+TOP_MARGIN`
So `380x5-20+44` → `width=380`, `origin=top-right`, `offset=20x44` (right margin x top margin).

### `shrink` — removed in v2

Remove entirely. `shrink = no` produces a WARNING and does nothing in v2.

### Safe v2 [global] keys for theming

```ini
[global]
    font                = JetBrainsMono Nerd Font 11
    markup              = full
    format              = "<b>%s</b>\n%b"
    width               = 380
    offset              = 20x44
    origin              = top-right
    transparency        = 10
    frame_width         = 2
    frame_color         = #primary-color
    corner_radius       = 8
    separator_color     = #surface-color
    separator_height    = 1
    padding             = 10
    horizontal_padding  = 14
    icon_position       = left
    max_icon_size       = 36
    sort                = yes
    indicate_hidden     = yes
    word_wrap           = yes
    follow              = mouse
    sticky_history      = yes
    history_length      = 20
    idle_threshold      = 120
    line_height         = 2
```

## DBus Conflict with Other Notification Daemons

Only one process can own `org.freedesktop.Notifications` on DBus at a time. If another notification daemon (KDE's plasmashell, GNOME's notification-daemon, mako, etc.) starts before dunst, dunst cannot acquire the name.

### Symptoms

- `notify-send "test"` exits 0 but nothing appears on screen
- Dunst logs: *"Cannot acquire 'org.freedesktop.Notifications': Name is acquired by '...' with PID XXXX"*

### Diagnose

```bash
# Check who owns the notification bus
dbus-send --session --print-reply --dest=org.freedesktop.DBus \
  /org/freedesktop/DBus org.freedesktop.DBus.GetNameOwner \
  string:org.freedesktop.Notifications

# Compare the PID
pgrep dunst
```

### Fix

Kill the competing daemon before starting dunst:

```bash
pkill dunst
dunst -config ~/.config/dunst/dunstrc &
sleep 1
notify-send "Dunst is working" "DBus reclaimed"
```

### Check active notification server

```bash
# gdbus (works on most systems)
gdbus call --session --dest org.freedesktop.Notifications \
  --object-path /org/freedesktop/Notifications \
  --method org.freedesktop.Notifications.GetServerInformation

# Or with qdbus6 (KDE systems)
qdbus6 org.freedesktop.Notifications /org/freedesktop/Notifications \
  org.freedesktop.Notifications.GetServerInformation
```

## Include Directive Version Check

Fragment includes require dunst >= 1.7.0. Verify:

```bash
dunst --version
```

On older versions, merge the fragment contents directly into dunstrc.
