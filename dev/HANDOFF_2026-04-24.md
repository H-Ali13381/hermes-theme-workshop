## The Design Stance Model

The pendulum between mainstream (Curated) and counter-culture (Liberated), with a second flavor axis (Warm/Cold × Meditative/Confrontational):

| Stance | Position | Flavor | Roots |
|--------|----------|--------|-------|
| **Zen** | Curated | Warm / Meditative | Apple HIG, wabi-sabi |
| **Signal** | Curated | Cold / Meditative | Material, Fluent, GNOME |
| **Garden** | Liberated | Warm / Meditative | Hippie, doujin, lo-fi |
| **Ghost** | Liberated | Cold / Meditative | Cypherpunk, GNU, tiling WM |
| **Riot** | Liberated | **Cold** / Confrontational | Punk, anarchist (visual language is cold — acid green/black, jagged bitmaps) |
| **Blade** | Liberated | Cold / Confrontational | Cyberpunk, hacktivist (precision-confrontational vs Riot's rough-confrontational) |
| **Drift** | Floating | Eclectic | Situationist dérive, chaotic neutral |

**Key corrected from earlier:** Riot is **Cold/Confrontational**, not Warm/Confrontational. Punk visual language (ransom-note, acid neon, jagged bitmaps) reads cold. The community ethos may be warm but the *aesthetic* is cold and adversarial.

**Blends:**
- Zen + Garden → Studio (warm, quiet, personal)
- Ghost + Blade → Hardened workstation
- Riot + Garden → Folk punk (rough dotfiles, warm prose)
- Signal + Ghost → Minimalist professional
- Blade + Garden → "Spaceship but cozy"
- Zen + Ghost → "Feels like a library"

Stances are **internal vocabulary** — agent uses them to orient itself, speaks to user in vibes not taxonomy.

## Session State & Persistence (NEW)

Every session creates `~/.config/rice-sessions/<theme-slug>-<YYYYMMDD-HHMM>/` with:

```
session.md       ← live log, one ## Step N section appended per completed step
design.json      ← written at Step 3
plan.html        ← written at Step 4
handoff.md       ← written at Step 8
handoff.html     ← written at Step 8
```

**Resume detection** runs at Pre-flight (before Step 1):
```bash
grep -rl "Status: IN PROGRESS" ~/.config/rice-sessions/*/session.md 2>/dev/null
```

`session.md` header: `Status: IN PROGRESS — Step N complete` → `Status: COMPLETE`

Critical rule: **write each step's section immediately on completion**, not at session end.

## Device Awareness in Step 1 (NEW)

Step 1 audit now includes a **Device Profile** block:
- Chassis type detection + battery check (laptop vs desktop)
- Screen count + resolution
- Touchpad presence
- GPU + VRAM

Decision table baked into Step 1:

| Device | Implication |
|--------|-------------|
| Laptop, single screen | Hyprland, minimal chrome, auto-hiding panels, keyboard-centric |
| Laptop + touchpad | Step 2 question: keyboard vs touchpad? → configure gestures if touchpad-heavy |
| Desktop, single screen | KDE or Hyprland both viable |
| Desktop, multi-monitor | Rich panels, per-monitor workspaces, KDE multi-monitor strong |
| Low VRAM | No blur/animations/animated wallpaper |
| High-end GPU | Full effects stack viable |
