# ricer-gtk — GTK Theming Layer

Part of the hermes-ricer suite. GTK is a completely separate rendering stack from Qt/KDE —
they do not share theme configs. Without this layer, a `void-dragon` Qt theme applied via
ricer leaves all GTK apps (Firefox, Nautilus, GIMP, most third-party apps) visually unchanged,
creating a jarring mismatch.

---

## What GTK Theming Controls

- Window decorations and titlebar chrome for GTK3/GTK4 apps
- Button styles, entry fields, checkboxes, scrollbars
- Menu backgrounds and text colors
- NOT: web page content, app icons, font rendering

---

## GTK Detection in discover_apps()

GTK has no single binary to check. Detection should be presence-based:

```python
gtk3_config = HOME / ".config" / "gtk-3.0"
gtk4_config = HOME / ".config" / "gtk-4.0"
# GTK is always present on a modern Linux desktop — register unconditionally
# (skip only if we have strong evidence GTK is not installed)
apps["gtk"] = {
    "installed": True,
    "gtk3_settings": str(gtk3_config / "settings.ini"),
    "gtk4_settings": str(gtk4_config / "settings.ini"),
    "gtk4_css": str(gtk4_config / "gtk.css"),
}
```

Add this block to `discover_apps()` in `ricer.py`, then register in `APP_MATERIALIZERS`:

```python
APP_MATERIALIZERS = {
    ...
    "gtk": materialize_gtk,
}
```

## settings.ini Format

`~/.config/gtk-3.0/settings.ini` and `~/.config/gtk-4.0/settings.ini` share the same format:

```ini
[Settings]
gtk-theme-name=Adwaita-dark
gtk-icon-theme-name=Papirus-Dark
gtk-font-name=Inter 11
gtk-cursor-theme-name=catppuccin-macchiato-teal-cursors
gtk-cursor-theme-size=24
gtk-toolbar-style=GTK_TOOLBAR_BOTH_HORIZ
gtk-button-images=1
gtk-menu-images=1
gtk-enable-event-sounds=0
gtk-enable-input-feedback-sounds=0
gtk-xft-antialias=1
gtk-xft-hinting=1
gtk-xft-hintstyle=hintslight
gtk-xft-rgba=rgb
gtk-application-prefer-dark-theme=1
```

Key fields to set from palette:
- `gtk-application-prefer-dark-theme=1` — always set for dark palettes
- `gtk-theme-name` — use an installed GTK theme that matches the palette mood
- `gtk-cursor-theme-name` — sync with cursor_theme from the KDE layer

## gtk.css for Fine-Grained Color Overrides (GTK4 only)

`~/.config/gtk-4.0/gtk.css` lets you override specific tokens with exact palette colors.
This is the most "corporate architecture" approach — no dependencies, version-controlled.

Minimal template:

```css
/* Generated by hermes-ricer */
@define-color accent_color {primary};
@define-color accent_bg_color {primary};
@define-color accent_fg_color {background};
@define-color destructive_color {danger};
@define-color success_color {success};
@define-color warning_color {warning};
@define-color error_color {danger};

/* Window chrome */
window {
  background-color: {background};
  color: {foreground};
}

/* Headerbars */
headerbar {
  background-color: {surface};
  color: {foreground};
  border-bottom: 1px solid {primary};
}

/* Entry fields */
entry {
  background-color: {surface};
  color: {foreground};
  border: 1px solid {muted};
}

entry:focus {
  border-color: {primary};
}

/* Buttons */
button {
  background-color: {surface};
  color: {foreground};
}

button:hover {
  background-color: {primary};
  color: {background};
}
```

Replace `{key}` with actual hex values from the palette at materialize time.

## materialize_gtk() Implementation

```python
def materialize_gtk(design: dict, backup_ts: str, dry_run: bool = False) -> list[dict]:
    """Materialize GTK 3 and GTK 4 theme settings from the active palette."""
    palette = design.get("palette", {})
    changes = []

    gtk3_dir = HOME / ".config" / "gtk-3.0"
    gtk4_dir = HOME / ".config" / "gtk-4.0"
    gtk3_settings = gtk3_dir / "settings.ini"
    gtk4_settings = gtk4_dir / "settings.ini"
    gtk4_css = gtk4_dir / "gtk.css"

    # Determine if dark palette (background luminance heuristic)
    bg = palette.get("background", "#1e1e2e")
    r, g, b = int(bg[1:3], 16), int(bg[3:5], 16), int(bg[5:7], 16)
    is_dark = (0.299 * r + 0.587 * g + 0.114 * b) < 128

    cursor_theme = design.get("cursor_theme", "Adwaita")
    gtk_theme = design.get("gtk_theme", "Adwaita-dark" if is_dark else "Adwaita")

    settings_content = f"""[Settings]
gtk-theme-name={gtk_theme}
gtk-icon-theme-name=Papirus-Dark
gtk-font-name={design.get('typography', {}).get('ui_font', 'Inter')} 11
gtk-cursor-theme-name={cursor_theme}
gtk-cursor-theme-size=24
gtk-button-images=1
gtk-menu-images=1
gtk-enable-event-sounds=0
gtk-enable-input-feedback-sounds=0
gtk-xft-antialias=1
gtk-xft-hinting=1
gtk-xft-hintstyle=hintslight
gtk-xft-rgba=rgb
gtk-application-prefer-dark-theme={1 if is_dark else 0}
"""

    p = palette
    css_content = f"""/* Generated by hermes-ricer — {design.get('name', 'theme')} */
@define-color accent_color {p.get('primary', '#89b4fa')};
@define-color accent_bg_color {p.get('primary', '#89b4fa')};
@define-color accent_fg_color {p.get('background', '#1e1e2e')};
@define-color destructive_color {p.get('danger', '#f38ba8')};
@define-color success_color {p.get('success', '#a6e3a1')};
@define-color warning_color {p.get('warning', '#f9e2af')};
@define-color error_color {p.get('danger', '#f38ba8')};

window {{
  background-color: {p.get('background', '#1e1e2e')};
  color: {p.get('foreground', '#cdd6f4')};
}}

headerbar {{
  background-color: {p.get('surface', '#313244')};
  color: {p.get('foreground', '#cdd6f4')};
  border-bottom: 1px solid {p.get('primary', '#89b4fa')};
}}

entry {{
  background-color: {p.get('surface', '#313244')};
  color: {p.get('foreground', '#cdd6f4')};
  border: 1px solid {p.get('muted', '#6c7086')};
}}

entry:focus {{
  border-color: {p.get('primary', '#89b4fa')};
}}

button {{
  background-color: {p.get('surface', '#313244')};
  color: {p.get('foreground', '#cdd6f4')};
}}

button:hover {{
  background-color: {p.get('primary', '#89b4fa')};
  color: {p.get('background', '#1e1e2e')};
}}
"""

    if dry_run:
        changes.append({
            "app": "gtk",
            "action": "dry-run",
            "gtk3_settings": str(gtk3_settings),
            "gtk4_settings": str(gtk4_settings),
            "gtk4_css": str(gtk4_css),
            "gtk_theme": gtk_theme,
            "cursor_theme": cursor_theme,
        })
        return changes

    # Backup existing files
    backup_file(gtk3_settings, backup_ts, "gtk/gtk3-settings.ini")
    backup_file(gtk4_settings, backup_ts, "gtk/gtk4-settings.ini")
    backup_file(gtk4_css, backup_ts, "gtk/gtk4.css")

    # Write GTK 3
    gtk3_dir.mkdir(parents=True, exist_ok=True)
    gtk3_settings.write_text(settings_content, encoding="utf-8")
    changes.append({"app": "gtk3", "action": "write", "path": str(gtk3_settings)})

    # Write GTK 4
    gtk4_dir.mkdir(parents=True, exist_ok=True)
    gtk4_settings.write_text(settings_content, encoding="utf-8")
    changes.append({"app": "gtk4", "action": "write", "path": str(gtk4_settings)})

    # Write gtk.css
    gtk4_css.write_text(css_content, encoding="utf-8")
    changes.append({"app": "gtk4-css", "action": "write", "path": str(gtk4_css)})

    return changes
```

## GTK Snapshot for Undo

Add `snapshot_gtk_state()` to ricer.py and call it from `snapshot_kde_state()` or a unified
`snapshot_desktop_state()` function:

```python
def snapshot_gtk_state() -> dict:
    """Capture current GTK theme settings for undo."""
    state = {
        "gtk3_settings": None,
        "gtk4_settings": None,
        "gtk4_css": None,
    }
    gtk3_path = HOME / ".config" / "gtk-3.0" / "settings.ini"
    gtk4_path = HOME / ".config" / "gtk-4.0" / "settings.ini"
    gtk4_css = HOME / ".config" / "gtk-4.0" / "gtk.css"
    if gtk3_path.exists():
        state["gtk3_settings"] = gtk3_path.read_text(encoding="utf-8")
    if gtk4_path.exists():
        state["gtk4_settings"] = gtk4_path.read_text(encoding="utf-8")
    if gtk4_css.exists():
        state["gtk4_css"] = gtk4_css.read_text(encoding="utf-8")
    return state
```

Undo: restore backed-up files from the backup dir (ricer.py already handles
file-based undo generically via `backup_file()` + restore on `undo()`).

## Known Pitfalls

- **`kde-gtk-config` is NOT installed on this system.** Do NOT rely on it.
  Write `settings.ini` files directly. KDE System Settings > "Legacy Application Style"
  tab will not function, which is fine — direct file writes are more reliable.

- **GTK theming does not control web page content, app icons, or font rendering.**
  Firefox uses its own internal theming on top of GTK chrome. The headerbar and menu
  bar pick up GTK colors; page content does not.

- **GTK 3 and GTK 4 use the same settings.ini format but different GTK themes.**
  A GTK3 theme (e.g. Catppuccin-Mocha-Teal) will not automatically apply to GTK4 apps.
  `gtk.css` is the correct GTK4 customization path.

- **GTK apps require restart to pick up settings.ini changes.** There is no live-reload
  equivalent for GTK — apps must be closed and reopened.

- **`gtk-application-prefer-dark-theme=1` only works for apps that respect it.**
  Well-behaved GTK4 apps honor this; older GTK3 apps may not.

- **Adwaita-dark is always available** as a safe fallback GTK theme — no install needed.
  For a closer palette match, Catppuccin GTK (https://github.com/catppuccin/gtk) can be
  manually installed to `~/.themes/`.
