# Terminal Theming — kitty & alacritty

## kitty

Config: `~/.config/kitty/`

### Include Injection vs Full Rewrite

If `kitty.conf` has hardcoded inline color definitions, the include approach **DOES NOT WORK** — kitty processes config top-to-bottom and inline values take precedence.

**Detection:**
```bash
grep -c "^color[0-9]\|^foreground\|^background" ~/.config/kitty/kitty.conf
```
If count > 5, the file is a hardcoded theme → **full rewrite** required.

**Include injection** (only for vanilla configs):
1. Write `~/.config/kitty/theme.conf` with all color assignments
2. Inject `include theme.conf  # linux-ricing` at end of `kitty.conf` (once only)
3. Back up both files before writing

**Full rewrite**: Back up entire kitty.conf, write new one preserving non-color settings (font, scrollback, tabs, window padding, opacity).

### Color Mapping

The canonical 10-key → 16 ANSI color mapping is defined in `shared/design-system.md`. Kitty and alacritty materializers use this mapping directly.

Additional kitty-specific mappings:

```
selection_bg     → primary  (yiq_text_color for selection_foreground)
cursor           → accent   (yiq_text_color for cursor_text_color)
url_color        → primary
```

### Live Reload

```bash
kill -SIGUSR1 $(pgrep -x kitty)
```

kitty picks up config changes via SIGUSR1 — no restart needed.

### Undo

Restore `theme.conf` from backup. Remove injected include line matched by `# linux-ricing` marker.

## alacritty

Config: `~/.config/alacritty/alacritty.toml` (v0.13+) or `alacritty.yml` (older)

Full rewrite on theming — alacritty has no include system.

### TOML Config Structure

```toml
[colors.primary]
background = "#0c1220"
foreground = "#e4f0ff"

[colors.cursor]
text   = "#0c1220"
cursor = "#d4a012"

[colors.normal]
black   = "#1c1e2a"
red     = "#cc3090"
green   = "#2a8060"
yellow  = "#c87820"
blue    = "#7ad4f0"
magenta = "#0d2e32"
cyan    = "#d4a012"
white   = "#e4f0ff"

[colors.bright]
black   = "#3d2214"
red     = "#d94ba5"
green   = "#35a578"
yellow  = "#d89030"
blue    = "#6ad0f5"
magenta = "#15483e"
cyan    = "#e0b525"
white   = "#f0f5ff"

[font]
size = 11.0

[font.normal]
family = "JetBrainsMono Nerd Font"

[window]
opacity = 0.92
padding = { x = 8, y = 8 }
```

### Live Reload

Alacritty watches its config file — changes apply to running instances immediately. No signal needed.

### YAML vs TOML Pitfall

Alacritty switched from `.yml` to `.toml` in v0.13. Check which exists:
```bash
ls ~/.config/alacritty/
```
If both exist, **TOML takes precedence**.

## wezterm

Config: `~/.config/wezterm/wezterm.lua` (Lua). WezTerm has no include system — the ricer **fully rewrites** the color scheme section using the `return` table.

### Config Structure

WezTerm uses a Lua table for config. The ricer patches the `colors` key:

```lua
-- Generated by linux-ricing — void-dragon
local wezterm = require 'wezterm'
local config = wezterm.config_builder()

config.colors = {
  foreground      = '#dae6f4',
  background      = '#203547',
  cursor_bg       = '#c69610',
  cursor_border   = '#c69610',
  cursor_fg       = '#203547',
  selection_bg    = '#c69610',
  selection_fg    = '#203547',

  ansi = {
    '#29445c',  -- black   → muted
    '#cc3344',  -- red     → danger synth
    '#3a9b5c',  -- green   → success
    '#d4a012',  -- yellow  → warning
    '#7ad4f0',  -- blue    → derived from primary hue
    '#c69610',  -- magenta → primary
    '#4c697a',  -- cyan    → surface
    '#dae6f4',  -- white   → foreground
  },
  brights = {
    '#4c697a',  -- bright black  → surface
    '#d94ba5',  -- bright red
    '#35a578',  -- bright green
    '#c87820',  -- bright yellow
    '#8ae0ff',  -- bright blue
    '#d4b030',  -- bright magenta
    '#5d7f90',  -- bright cyan
    '#ffffff',  -- bright white
  },
}

return config
```

### Ricer Approach

WezTerm's Lua config can be complex. The ricer:
1. Detects if `wezterm.lua` exists
2. If a `config.colors = {` block is present — replaces it in-place using markers
3. If not — appends the full `colors` block before the final `return config`
4. **Does not modify** font, window, keybinding, tab settings — only `colors`

### Live Reload

WezTerm watches its config and reloads automatically — no signal or restart needed.

### Pitfall

If the config uses `wezterm.color.get_builtin_schemes()` or `color_scheme = "SomeName"`, those **override** the `colors` table. Check and remove `color_scheme` assignments before apply.

## foot

Config: `~/.config/foot/foot.ini` (INI). foot has no include system — the ricer **fully rewrites** the `[colors]` section, preserving all other sections.

### Config Structure

```ini
[colors]
# Generated by linux-ricing — void-dragon
background=203547
foreground=dae6f4

# Note: foot uses hex WITHOUT the # prefix
regular0=29445c  # black  → muted
regular1=cc3344  # red    → danger
regular2=3a9b5c  # green  → success
regular3=d4a012  # yellow → warning
regular4=7ad4f0  # blue
regular5=c69610  # magenta → primary
regular6=4c697a  # cyan   → surface
regular7=dae6f4  # white  → foreground

bright0=4c697a
bright1=d94ba5
bright2=35a578
bright3=c87820
bright4=8ae0ff
bright5=d4b030
bright6=5d7f90
bright7=ffffff
```

### Critical Pitfall — No `#` Prefix

**foot uses hex colors WITHOUT the `#` prefix.** `background=203547` is correct; `background=#203547` is a parse error and foot silently falls back to defaults. The ricer strips `#` from all palette values before writing the foot config.

### Ricer Approach

1. Read `~/.config/foot/foot.ini`
2. Replace the entire `[colors]` section content (from `[colors]` to the next `[section]` or EOF)
3. Back up the original before writing
4. Reload: `kill -SIGUSR1 $(pgrep foot)` — foot reloads config for all open instances

### Live Reload

```bash
kill -SIGUSR1 $(pgrep foot)
```

foot reloads config on SIGUSR1. All open windows update immediately.

## Display Fonts (Aesthetic Extreme)

Standard ricing uses monospace fonts. For fantasy/gothic themes, display fonts create total coherence:

```ini
# kitty.conf
font_family      Metamorphous
bold_font        Metamorphous
italic_font      Metamorphous
font_size        10.0
```

Good candidates (Google Fonts / Nerd Fonts):
- **Metamorphous** — medieval serif, best for gothic/dark fantasy
- **Cinzel** — Roman serif, classical or arcane
- **Uncial Antiqua** — insular manuscript style
- **MedievalSharp** — blackletter-influenced
- **Press Start 2P** — pixel/retro game aesthetic

**Pitfall:** Display fonts have no bold/italic variants. Set `bold_font` and `italic_font` to the same family to prevent fallback to system monospace.
